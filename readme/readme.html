How to Website

So you are the webmaster, what do you do

Everyone gives you stuff and you add it to a JSON file. That's all your duties, other than working with the other officers, so have fun!

How do I view the site?

Because stuff is loaded from a JSON file, you have to run some sort of server to view the games page.

One easy way is open a terminal, navigate with cd to the directory, and run python3 -m http.server

Updating the Games List (games.json)

Looking at the existing games in the file should give you a good idea of how to add a game.

The file is split into tabs (for the years), which contain groups of games.

Tabs can be in any order, because they are automatically sorted by year and then alphabetically. The site will always open to the most recent year.

Whitespace doesn't matter, so you can space out the properties however you want to make it more readable.

List of game properties

Properties can be listed in any order. Remember to put a comma between each property.

  • name: The title of the game. The only required property.
  • tagline: A short description of the game.
  • description: A longer description of the game.
  • engine: The name of the engine that game was created in.
  • images: A list containing links/file paths to images of the game. They are displayed in the order you write them in.
  • links: A list containing link information. They are displayed in the order you write them in. You can add the name of a material design icon if you'd like your link to have a different icon next to it.
  • roster: List of people who worked on the game (credits).

Debugging

  • You remembered to use a server and didn't just double click the file
  • There are commas between every entry in the lists
  • There are commas between every property
  • There are commas between every tab
  • All brackets/braces of any kind have a matching end bracket/brace

Using some sort of JSON editor or validator may be useful.
  • You saved the file
  • You reloaded the page
  • The property name is spelled correctly
  • Everything that should be in quotes actually has quotes around it
  • Your content is inside the list and not before or after
  • If the problem is that the online website doesn't reflect the changes, make sure you actually committed and pushed your changes

You can always duplicate an existing game entry to use as a template.

Using the components

Components are basically custom html elements, included through scripts. Once you include the template script, they can be used in the #gamedev-root element.

The source files for the components are found in the "components" folder, and they all start with the prefix "gamedev-". Check them out if you're familiar with Vue or just want to look at the html templates and css classes.

To use components, you need 4 things:

  • Include Vue: The components require Vue to work, so makes sure the page has included vue with:
  • Include the template component: The template component creates a Vue app (necessary to use components). It also includes the header and footer components; they don't need to be included separately.
  • Include the script of the component you want: Each component is contained in a js file that has to be included. Check the full list of which components are in which file.
  • Start up the app!

Why didn't you use webpack/vite/something else?

My main goal was to make the website as easy to edit as possible for someone with knowledge of only html5/css3. With this type of component registration, whoever is maintaining the website just has to include the scripts and then write html with a few extra elements, and there is (hopefully) no confusion over scoped css or the framework itself. This also uses Options API rather than Composition API as it's closer to what is taught in required UMBC CS classes.

Additional notes

If you end up inserting content using Vue's "mustaches", remember to include v-cloak on the element so the content won't show until Vue has loaded.
